# Completed Requests

The log. One planning doc per person per day, sitting next to the files that produced it.

```
2026-07-15-jacob-maddux.md          <- what was asked for, and what got built
2026-07-15-power-grid-notes.txt     <- the original request, untouched
2026-07-15-mockup.png
```

Each doc holds the request as Claude understood it, the slices it was broken into, every objective
ticked off, and a closing note covering what couldn't be verified. Nothing here was compiled by the
machine that wrote it — a tick means "built and reviewed", not "builds".

Read `- [!]` items. They're objectives Claude deliberately did **not** build, with the reason
underneath — usually a conflict in the request or something that would have broken elsewhere.

**Don't hand-edit these.** If the same person submits again on the same date, Claude moves that day's
doc back to `in-progress/`, appends the new objectives, and files it here again when it's done — so the
day stays one record rather than fragmenting across uploads. (The earlier request files stay put here
while that happens; only the doc travels.)

Empty right now. It fills up as requests land.

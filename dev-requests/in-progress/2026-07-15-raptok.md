## What they asked for

A large overhaul of the planet UI. Two parts.

**Part 1 — retire the redundant/deprecated windows that pop up when you select a planet.** Four
windows are named, all of which the newer Planet View UI has superseded:

1. The **bottom-left "Planet Info Panel"** — old UI, all its info now lives in the Planet View. Remove it.
   *(In code this is `PlanetUI`'s `infoPanel`. `PlanetUI` itself is the selection hub — it fires
   `OnBodySelected`, focuses the camera, floats labels — so only the visible panel goes, not the hub.)*
2. The **"Around Homeworld"** window — a scrollable list of the planet's moons you click to hop between
   bodies. No longer needed. *(In code: `AssociatedObjectsWindow`.)* Its job is replaced by the moon tabs.
3. The **"Colony — Homeworld"** window — buttons to upgrade the Shipyard / Research Centre, build
   Farms/Mines, etc. Most of that has moved to surface Build Mode; the **Shipyard** part should move to
   the Planet View's new **Orbit** tab. *(In code: `ColonyWindow`.)*
4. The **"Homeworld"** tabbed window (Overview / Climate / Ores / Society / Production / Objects /
   Terraform). Fold each tab into the new Planet View, then stop it appearing. *(In code:
   `InspectorWindow` showing an `InspectorBodyTabs` body.)* Mapping they gave:
   - Overview → Overview tab
   - Society, Production → Overview tab
   - Climate, Ores, Terraform → Survey Mode tab
   - Objects (things in orbit) → Orbit tab

**Part 2 — restructure the Planet View UI.**

- Clicking a planet fills the screen with the Planet View.
- The surface **Map anchors to the left**, with room above it for the tabs and below it for other UI.
  The map must never exceed **3/4 of the screen width**; the far-right **1/4** holds the tab's panel.
- Top tabs become exactly: **Overview, Build Mode, Survey Mode, Orbit**. Existing Build/Survey
  functionality moves onto their tabs; the selected tab's content fills the panel right of the map.
- **Moon tabs** on the underside of the map, one per moon, stacked with the closest moon leftmost.
  Clicking one shrinks the host map down from the top to make room, and shows the moon's own grid map
  above the host, centred on the host map's centreline. A second moon opens to the right of the first
  and pushes it left, so the pair straddles the host's centreline; closing one recentres the other.
  Grids get spacing so they read as separate.

### How I sliced it, and honest scope note

This is a very large request that essentially rewrites the planet UI, and **this project cannot be
compiled in CI (Unity, no Editor)** — so every slice is reviewed by subagents rather than built. I have
ordered slices by dependency and risk, safest first, and folded functionality into the Planet View
*before* retiring the window that owned it (so nothing is stranded). The moon dual-map feature (Slice 6)
is the largest and riskiest; if the budget runs out it stays `- [ ]` for the next run.

Ambiguities I resolved: "Research Centre" is a ground facility, so its upgrade stays reachable from the
colony/Overview side rather than the Orbit tab; only the **Shipyard** (a space construct) moves to Orbit
as they specified. "Objects in orbit" I read as ships docked/in orbit plus orbital constructs.

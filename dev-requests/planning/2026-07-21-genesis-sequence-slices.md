# Genesis Sequence & Object Visibility — slice plan

**Source:** Jacob's "Genesis Sequence & Object Visibility — Full Build Brief" (pasted 2026-07-21).
**Status:** In progress — slices 1 and 2 built.

## The one correction the brief needs

The brief says "nothing here has been built yet — treat it as a specification, not a status report."
That is not accurate, and building against it literally would mean rewriting working code. What
actually existed on 2026-07-21:

| Brief section | Reality |
|---|---|
| Part 2, the genesis sequence | **Largely built**, but as a *private preview stage* rendered to a RenderTexture and shown in a RawImage — real home star(s), binary/trinary pop-out, homeworld terrain, real moons, bar collapse, travel to centre, welcome text, cross-fade handoff. The brief's governing idea (*the loading screen IS the live camera view of the real world*) is precisely what it is not. |
| Part 3, terrain generation | CPU-baked (`SurfaceTextureRenderer` → `Texture2D`). Tiles **are** gameplay data (`SurfaceBuildManager` places footprints on `body.surface`) — that answers the brief's open question #2. |
| Part 4, the backdrop | **Built** — `SpaceBackground`: nebulae, coloured twinkling stars, shooting stars, parallax. |
| Part 6, orbit lines | **Built** — blue `LineRenderer` rings with a global reveal fade already used as the "you have control" cue. |
| Part 1, visibility | **Genuinely absent.** Built in slice 1. |
| Part 7.3, float precision | **Not a problem.** The galaxy is ~6,400 units across; float degrades past ~10⁵. Four orders of magnitude of headroom — a floating origin would be wasted work. |

## Slices

### 1. Universal visibility, hiding and deletion — DONE
- [x] `HideReason` (Dev · Cloaked · Undiscovered · Sequence) on bodies, stars, systems and ships
- [x] `VisibilityService` — the single place concealment is decided
- [x] `ConcealBinding` — disables renderers/colliders/lights, restores exactly what it disabled
- [x] Orbit lines, owner rings, habitable rings, galaxy-zoom proxies, derelict hulls, name labels
- [x] `GalaxyTrash` — real deletion with a restoring bin; homeIndex and derelict indices fixed up
- [x] `ObjectVisibilityWindow` — the Dev-Mode tree that makes it usable
- [x] Rare naturally-hidden worlds at generation (~1.2%)
- [x] Save/load persistence, with `Sequence` deliberately never written to disk

### 2. The galaxy arrives (Beat 9) — DONE
- [x] `GenesisReveal` — galaxy concealed at generation, homeworld revealed when the camera takes
      over, the rest arriving a beat after the orbit lines
- [x] Backstop in `GenerateGalaxyRoutine`'s `finally`, not at the end of the body

### 3. Terrain as a transformation, not a reveal — IN PROGRESS
- [x] Ten development stages per world, generated by lerping `terrainParams` from a primordial state
      toward the real ones — the brief's own instinct ("the same mechanics as the terrain sliders,
      with the panel hidden") turned out to be the right one
- [x] The last stage IS the real generated surface, so what the world settles into is exactly what
      gets played on
- [x] Moons get the identical treatment
- [ ] Verify in-editor: does a wet world visibly grow continents out of a global ocean?

### 4. Tile legibility at true relative scale — NOT STARTED
The brief's own §5.4 conflict. At the dollar-coin anchor a moon is ~25–43 px across, too small for a
tile lattice to read as tiles.
- [ ] Scale tile COUNT by apparent on-screen size (brief's option (a)) — keeps true relative body
      size, which option (b) would break
- [ ] Small camera ease-in during the moon beat (option (c))

### 5. Scale and framing spec — NOT STARTED
- [ ] One serialized config: homeworld ≈ 9% of viewport height at ~35% of viewport width
- [ ] ONE compression curve for relative sizes (star ≈ 1.5–2.5× the homeworld, moon ≈ 0.25–0.45×
      its planet), derived in one place so "consistently relative to each other" actually holds
- [ ] 1.0× → 1.3× → 1.0× across the closing beats, as tunable constants

### 6. The camera-driven sequence — NOT STARTED, LARGEST
Replaces the RenderTexture preview with the real camera over the real world.
- [ ] Screen-space anchoring (the maths already exists in `HandoffScreenFraction`)
- [ ] Exclusive camera ownership during the sequence, clean handback at Beat 10
- [ ] Loading bar bound to real generation progress (already true — the generator is phased)
- [ ] Skip/abort; never on loading a save
**Note:** this is the only slice that replaces working, tuned code rather than adding to it. Worth
doing last, and worth confirming it is wanted before starting.

### 7. Orbit conveyed at close framing — NOT STARTED
The brief's §5.5, and it is a real gotcha: when the camera tracks the homeworld, "it is orbiting its
star" reads as completely static, because there is no relative motion on screen.
- [ ] Starfield parallax as it travels (this is why Part 4's parallax matters beyond aesthetics)
- [ ] Terminator sweep across the surface
- [ ] Slight camera drift

## Notes

**Nothing in this repo is compiled by the agent that writes it** — there is no Unity in that
environment. Every slice is reviewed hard by independent agents instead, and every one so far has
turned up real bugs. Build before playing.

**Automation:** this repo already has it. Dropping a request file in `dev-requests/in-progress/` and
pushing triggers the *Dev Requests* GitHub Action, which reads it, slices it, builds, reviews and
pushes back — see `dev-requests/in-progress/READ THESE INSTRUCTIONS FIRST.md`. It bills the
`ANTHROPIC_API_KEY` secret, so it is deliberately not wired up from here without asking.
